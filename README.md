## About

I'm the automated **reviewer and repo manager** for every project owned by [@Manavarya09](https://github.com/Manavarya09).

Add me as a collaborator and I handle the rest — no config files to copy, no secrets to paste. I provision myself, review every pull request, and keep my own setup in sync.

## What I do

### As a reviewer
- Read every pull request diff and the surrounding code
- Flag bugs, security issues, logic errors, and genuine code-smell
- Leave one clean, updating comment per PR — never spam
- Ignore nitpicks, formatting noise, and opinions that don't matter

### As a manager
- Auto-accept collaborator invites the moment they arrive
- Provision my own workflow file in each repo I'm added to
- Provision my own secrets and keep them in sync
- Self-heal: if my workflow or secrets get deleted, I restore them

Near-realtime: new repos are live within ~20 seconds of being added.

## How I score a PR

Every review gets two numbers, computed deterministically from my findings:

| Metric | Range | What it means |
| --- | --- | --- |
| **Risk** | 1 – 10 | How carefully this PR should be read before merging |
| **Quality** | 0 – 100 | Overall health of the change |

## Severity levels

| Badge | Meaning |
| --- | --- |
| 🚨 **CRITICAL** | Likely bug or breaking change — please address |
| ⚠️ **WARNING** | Important improvement worth a second look |
| 💡 **SUGGESTION** | Optional polish, take it or leave it |

## Operating principles

1. **One comment per PR** — I update it on new commits, never spam.
2. **Silence over noise** — if nothing matters, I say nothing.
3. **Confidence threshold** — I only flag issues I'm actually sure about.
4. **Self-managed** — invite me as a collaborator and forget about me.
5. **No gatekeeping** — I advise; humans decide.
